## 1. 项目背景与赛题对应

大语言模型在辅助论文写作时经常"一本正经地胡说八道"——推荐出格式专业、
实则根本不存在的参考文献。已有研究对 10 款商用大模型近 7 万次引用实例的测试显示，
虚假文献比例最低 11.4%、最高达 56.8%。本项目是针对赛题
《AI 推荐的文献是真是假？——学术文献引用智能验证智能体设计》的完整实现。

**赛题三项任务与本项目的对应关系：**

| 赛题任务 | 本项目实现 |
|---|---|
| 任务一：虚假文献特征分析与规律发现 | 「虚假特征分析」功能：按 生成模型 / 学术领域 / 主题 分组统计虚假率、每类特征命中条数、Top 失效规则；配合「数据画图」输出统计图表 |
| 任务二：可计算的验证规则体系 | 12 条加权规则（元数据格式校验 / 外部数据库验证 / 语义一致性三个维度），0–100 可信度评分模型，阈值 ≥80 可信、40–79 可疑、<40 虚假（与赛题示例一致）；规则文档见《文献引用可信度可计算验证规则体系.docx》与 `config/rules_default.yaml` |
| 任务三：可运行的智能体原型（必做） | 本 Streamlit 应用：单条验证（支持仅标题或仅 DOI 输入）+ 批量验证（加分项）+ Web 可视化界面（加分项），输出判定 / 分数 / 问题明细 |

> 验证核心（单条 / 批量 / 虚假特征 / 导出）**无需任何 API Key**，
> 仅用本地规则 + 公开学术 API 即可运行；DeepSeek 仅用于"智能问答 / AI 解读 /
> 按问作答的统计总结 / 自然语言画图"等增强功能。

## 2. 功能总览

应用是一个**聊天式智能体**：没有菜单页签，直接把需求发给它，
系统自动推断该调用哪个工具（文件类型优先，其次 LLM 意图分类 + 关键词规则兜底）。

| 功能 | 触发方式 | 说明 |
|---|---|---|
| 📋 单条验证 | 粘贴任意格式的引用文本（或仅 DOI / 标题） | 解析 → 查询 9 个学术库 → 12 条规则打分 → 判定卡片（判定 / 分数 / 关键证据 / 修复建议）+ 规则评分明细 + 外部证据表 |
| 📊 批量验证 | 上传 CSV / TSV / Excel / JSON / Parquet | 自动识别字段列（中英文表头均可），8 线程并发逐条核验，**实时进度条显示 已验条数 / 预计剩余时间**；结果回填中文列，可下载 CSV |
| 🔬 虚假特征分析 | 输入"统计虚假特征 / 各学科虚假率"等 | 按模型 / 领域 / 主题分组的虚假率、每类特征命中条数；**基于统计数据针对你的具体问题作答**，而非固定模板 |
| 📈 数据画图 | 输入"画个饼图 / 柱状图…" | DeepSeek 生成 Plotly 绘图代码，在受限沙箱中执行（无 import / 文件 / 网络，20s 超时，失败自动重试一轮）；可查看生成代码与作图数据 |
| 💬 智能问答 | 直接提问 | 流式回答（思考过程可折叠查看），回答基于本轮已有的核验数据，可追问"第几条为什么可疑"等 |
| 📄 报告导出 | 输入"导出报告" | 基于最近一次批量结果生成 HTML / PDF 报告下载 |

**批量验证 + 附带问题**：上传文件时在输入框同时写上问题（如"统计各学科的虚假文献率"），
验证完成后的 AI 总结会**优先回答该问题**（用表格），而非输出通用报告。

## 3. 安装与启动

### 环境要求

- Python **3.10 – 3.13**（推荐 3.12）
- 可访问公网（需调用 CrossRef / OpenAlex 等学术 API）

### Windows（一键，推荐）

```bat
run.bat
```

首次运行会自动完成全部准备（创建 `.venv` → 安装依赖 → 启动并打开浏览器）。
注意事项：

- 安装脚本会**自动探测**本机 Python（依次尝试 py 启动器的 3.12 / 3.11 / 3.13 / 3.10 与 PATH 中的 python，并真实试运行验证，可识破 Windows 商店的占位 python）；
- 依赖默认走**清华 PyPI 镜像**（国内直连 pypi.org 常超时卡死）。如需换源，先设置环境变量 `PIP_INDEX_URL` 再运行；
- 从其他电脑整体拷贝来的 `.venv` 不可用（虚拟环境不可移植），脚本会自动探测并原地重建；
- 启动后弹出的**黑色命令行窗口不要关闭**——它就是后端进程，关了应用即停止。

### macOS / Linux

```bash
./setup.sh    # 首次：建 .venv 装依赖（同样默认清华镜像、自动探测 Python）
./run.sh      # 启动
```

### 手动安装（任意平台）

```bash
python -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -r requirements.txt
.venv/bin/python -m streamlit run app.py
```

浏览器打开 `http://localhost:8501`。

### Docker

```bash
docker build -t litverify .
docker run -p 8501:8501 --env-file .env litverify
```

## 4. 配置说明

所有配置均可选。复制 `.env.example` 为 `.env` 按需填写，
或在应用内「⚙ 设置」临时填写 DeepSeek Key（仅当次会话有效）。

| 变量 | 作用 | 默认 |
|---|---|---|
| `DEEPSEEK_API_KEY` | 启用智能问答 / AI 解读 / 自然语言画图 / 按问作答总结 | 空（相关功能显示配置提示，核心验证不受影响） |
| `DEEPSEEK_BASE_URL` | DeepSeek API 地址 | `https://api.deepseek.com` |
| `DEEPSEEK_PRO_MODEL` / `DEEPSEEK_FLASH_MODEL` | Pro / Flash 两档模型名（侧边栏可切换；Pro 更聪明，Flash 更快） | `deepseek-v4-pro` / `deepseek-v4-flash` |
| `DEEPSEEK_CHAT_MODEL` | 默认使用哪一档 | `deepseek-v4-pro` |
| `CROSSREF_EMAIL` / `OPENALEX_EMAIL` | 学术 API 礼貌访问邮箱（建议填写，可获得更稳定的速率） | 空 |
| `SEMANTIC_SCHOLAR_API_KEY` / `NCBI_API_KEY` | 提升对应数据源的速率限额 | 空（匿名访问） |
| `WANFANG_APP_KEY` / `WANFANG_APP_SECRET` | 启用万方（中文文献覆盖），不填则跳过该源 | 空 |
| `API_TIMEOUT` | 单个学术 API 请求超时（秒） | 15 |
| `REAL_THRESHOLD` / `SUSPICIOUS_THRESHOLD` | 判定阈值 | 80 / 40 |
| `HISTORY_DB_PATH` | 历史记录 SQLite 路径（相对路径以项目根目录为锚） | `data/litverify_history.sqlite3` |
| `DUCKDB_THRESHOLD_MB` | 超过此大小的 CSV 改用 DuckDB 流式读取 | 100 |

> 注：DeepSeek v4 系列为**推理模型**——思考 token 计入 `max_tokens` 且先于正文输出。
> 项目内所有调用点均已按"思考 + 正文"预留足够预算，自行改代码时请注意这一点。

## 5. 使用指南

### 5.1 单条验证

直接把引用粘贴到输入框发送即可，支持完整引用、仅 DOI、仅标题三种形式：

| 输入 | 预期 |
|---|---|
| `doi:10.1038/nature14539` | 可信（CrossRef 命中 LeCun《Deep learning》） |
| `Vaswani A. Attention is all you need. NeurIPS, 2017.` | 可信 / 可疑（标题命中，无 DOI 稍弱） |
| `Smith J. Quantum hyperdrive learning. Nature, 2024. doi:10.1038/fake.99999` | 虚假（DOI 无法解析） |

### 5.2 批量验证（赛题测试数据）

仓库同级目录 `测试数据/文献数据(1).xlsx` 为赛题样例
（200 条，含 生成模型 / 学术领域 / 有关主题 / 完整标题 / 作者姓名 / 期刊会议 / 发表年份 / DOI 列）：

1. 点输入框左侧「＋」选择文件，或直接拖拽到页面；
2. （可选）在输入框写上你关心的问题，如"统计各学科的虚假文献率"；
3. 回车发送——**你的消息气泡与进度条会立即显示**，等进度条走完即可
   （200 条约 5–8 分钟，见 [第 9 节](#9-性能说明批量验证为什么需要几分钟)；期间请勿刷新页面或重复发送）。

**输出**：在原表上回填 `验证结果`（可信/可疑/虚假）、`可信度分数`（0–100）、
`虚假特征`（未通过项描述）、`命中DOI`、`命中标题` 列；
界面展示判定计数、占比饼图、得分分布直方图，可下载结果 CSV。

### 5.3 追问与统计（省时技巧）

**数据验证过一次后，追问不要再附文件**——直接打字提问，系统基于本轮已有结果作答，
几秒到十几秒出结果；重新附文件会把整批数据从头再验一遍。

- "统计虚假文献的常见特征的每个特征的条数" → 特征条数表
- "不同学科虚假文献率分布情况" → 按领域的虚假率
- "为什么第 1 条被判为虚假？" → 基于该条的规则得分与外部证据解释
- "画一个虚假特征频次的横向条形图" → 沙箱生成图表（提问时把统计口径说清楚，
  例如"以虚假样本数为分母"，可避免不同图表口径不一致）

### 5.4 报告导出

输入"导出报告"即可下载 HTML / PDF 版验证报告（基于最近一次批量结果，
无批量结果时回退到本机历史记录）。

## 6. 验证规则体系（任务二）

综合得分为各规则得分（0–1）按权重加权归一后的 0–100 分；
默认阈值 **≥80 可信、40–79 可疑、<40 虚假**（与赛题示例阈值一致，可在设置中调整）。

| 维度 | 规则 | 权重 |
|---|---|---|
| 外部数据库验证 | DOI 在外部 DOI / 学术库可解析（`doi_resolve`） | 20 |
| 外部数据库验证 | 标题与权威库一致，相似度 ≥0.85（`title_match`） | 18 |
| 外部数据库验证 | 作者列表一致，重合率 ≥0.70（`author_match`） | 12 |
| 外部数据库验证 | 期刊/会议存在且一致（`venue_match`） | 10 |
| 外部数据库验证 | ArXiv ID 可解析（`arxiv_resolve`） | 8 |
| 外部数据库验证 | 卷/期/页一致（`page_volume_consistent`） | 6 |
| 外部数据库验证 | 多源外部库一致（`cross_db_consistent`） | 6 |
| 元数据格式校验 | DOI 格式正则（`doi_format`） | 10 |
| 元数据格式校验 | 作者姓名格式合理（`author_format`） | 4 |
| 元数据格式校验 | 期刊/会议名称合理（`venue_format`） | 4 |
| 语义一致性 | 年份在 1900–2026 合理范围（`year_reasonable`） | 6 |
| 语义一致性 | 标题长度 6–300 且无异常字符（`title_quality`） | 4 |

- 完整定义见 `config/rules_default.yaml`；应用内「⚙ 设置 → 验证规则」可**现场开关规则、
  调整权重、导入/导出 YAML**，阈值同样可调；
- 设计依据与实验论证见随附文档《文献引用可信度可计算验证规则体系.docx》。

**为什么外部验证权重远高于格式校验？** 对赛题 200 条样例的实测显示：
大模型编造的 DOI 有 97%+ 格式完全规范（真实期刊前缀 + 编造后缀），
格式校验抓不住它们；只有拿 DOI / 标题去权威库实际解析比对才能识破——
这正是任务一特征分析得出的核心规律。

## 7. 外部学术数据源

| 数据源 | 用途 | 是否需要凭证 |
|---|---|---|
| CrossRef | DOI 解析 + 标题检索（主力） | 否（建议填邮箱） |
| OpenAlex | DOI / 标题检索（主力） | 否（建议填邮箱） |
| arXiv | arXiv ID 解析、预印本检索 | 否 |
| DataCite | 数据集类 DOI | 否 |
| DOIDB | 中文 DOI 解析 | 否 |
| PubMed | 医学文献（DOI 未命中时按标题兜底） | 否（NCBI Key 可提速率） |
| Semantic Scholar | 综合检索（兜底） | 否（Key 可提速率） |
| DBLP | 计算机科学文献（兜底） | 否 |
| 万方 | 中文文献 | **需要** `WANFANG_APP_KEY/SECRET`，未配置自动跳过 |

查询策略：先用 DOI 直接解析；DOI 未命中再按标题在各兜底源检索；
HTTP 结果带磁盘缓存（diskcache），重复查询不重复请求。

## 8. 技术架构与目录结构

```
app.py                  # Streamlit 入口（单页聊天式智能体）
config/
  settings.py           #   .env 配置加载、版本号
  prompts.py            #   DeepSeek 提示词（解析/解读/总结/画图/意图分类）
  rules_default.yaml    #   12 条规则的默认权重与阈值
services/
  agent_router.py       #   智能体路由：意图推断 → 工具分发 → 结果组装
  citation_parser.py    #   引用解析（正则 + LLM 摘录式解析，禁止补全幻觉）
  citation_verifier.py  #   单条验证编排（多源查询 → 规则引擎）
  rule_engine.py        #   12 条规则实现与加权评分
  data_processor.py     #   批量验证（8 线程并发 + 进度回调）
  fake_analyzer.py      #   任务一统计：分组虚假率 / 特征条数
  exporter.py           #   HTML / PDF 报告生成
  *_client.py           #   9 个学术数据源客户端（统一重试/超时/缓存基类）
llm/
  deepseek_client.py    #   DeepSeek 封装（普通 + 流式，含 reasoning 通道）
  insight_generator.py  #   单条验证的 AI 解读（失败回退本地解释）
ui/
  chat_shell.py         #   聊天外壳：消息流 / 组合器 / 流式渲染 / 批量进度条
  sidebar.py            #   会话管理侧边栏（底部显示版本号）
  settings_dialog.py    #   设置弹窗（主题 / 模型 / 阈值 / 规则编辑器）
  verdict_card.py       #   判定卡片
  theme.css / theme.py  #   液态玻璃视觉主题
utils/                  # DOI 工具 / 文本相似度 / 缓存 / 会话持久化
db/                     # SQLite：验证历史 + 聊天会话持久化（刷新不丢对话）
tests/                  # 63 个单元/集成测试
```

关键设计：

- **纯函数式工具分发**：`agent_router.dispatch(mode, text, files, history)`
  返回可序列化的消息 dict，UI 重跑时直接回放，不重复执行；
- **两段式执行**：发送后先落地用户气泡再执行长任务，避免"看起来没发出去"；
  流式回答被打断时自动保存已生成部分；
- **沙箱画图**：生成代码在无 import / 无文件 / 无网络的受限命名空间执行，
  静态令牌黑名单 + 20 秒超时双保险；
- **LLM 解析防幻觉**：引用解析只允许"摘录"原文出现的字段，
  原文没有的信息一律置空，杜绝模型按记忆"补全"出不存在的元数据。

## 9. 性能说明：批量验证为什么需要几分钟

每条引用要实打实地查询多个外部学术数据库（DOI 解析 → 未命中再按标题逐库检索）。
**虚假引用恰恰是最慢的**：假 DOI 永远解析不到，每条都要把兜底源问完才能下结论；
真实引用 DOI 一击命中反而快。

实测参考（8 线程并发）：

| 数据规模 | 耗时（本地） |
|---|---|
| 10 条 | ~20 秒 |
| 20 条 | ~35 秒 |
| 200 条（赛题样例，71.5% 为假） | **约 5–8 分钟** |

进度条会实时显示 已验条数 / 预计剩余时间。演示场景建议用 20–50 行子集，一两分钟跑完效果相同。
如需提速可上调 `data_processor` 的 `max_workers`（有被 CrossRef 限流的风险）或裁剪兜底数据源。

## 11. 常见问题 FAQ

**Q：发消息一直没有回应？**
先看页面右下角是否有灰色 "Connecting..."——那说明后端已停止（本地：黑色命令行窗口被关了；
线上：应用休眠），旧页面只是残影。本地重新运行 `run.bat`，线上点 "get this app back up"。
正常状态下，回车后你的气泡和进度条/加载指示会**立即**出现。

**Q：怎么确认线上/本地跑的是最新版本？**
展开左侧边栏，底部"DeepSeek 已连接"旁的小字就是版本号（如 `v2026-07-08.5`）。

**Q：批量验证的虚假率为什么这么高（70%+）？**
这是数据本身的属性，不是误判。赛题样例是 AI 生成的参考文献，对其中 30 条 DOI
直接请求官方 doi.org 解析服务实测：21/30（70%）返回 404——与系统判出的 71.5%
虚假率一致。典型造假模式是"真实期刊 DOI 前缀 + 编造后缀"。

**Q：真实的中文文献会被误判吗？**
无 DOI、仅收录于知网/万方的真实中文文献可能落入"可疑"档（西文库查不到），
建议对可疑档人工复核；配置万方 Key 可改善中文覆盖。

**Q：依赖安装很慢 / 卡住？**
国内直连 pypi.org 常超时。`setup.bat` / `setup.sh` 已默认走清华镜像；
如仍慢，检查代理设置，或设置 `PIP_INDEX_URL` 换其他镜像。

**Q：两张统计图的数字对不上？**
AI 画图是每次现写统计代码，口径可能不同（"提及次数占比" vs "样本条数"）。
每张图下方可展开「查看生成的 Python 代码」核对口径；提问时把分母说清楚即可获得一致口径。

## 12. 云端部署

本应用可直接部署到 [Streamlit Community Cloud](https://streamlit.io/cloud)：

1. Fork / push 本仓库到 GitHub（应用文件需位于仓库根目录）；
2. Streamlit Cloud 新建应用，入口 `app.py`；
3. 在应用 Settings → Secrets 配置 `DEEPSEEK_API_KEY` 等环境变量
   （云端共享部署会自动禁用聊天记录的本地持久化，避免访客间串话）；
4. 之后每次 push 到部署分支即自动重新部署，部署结果以侧边栏版本号为准。
